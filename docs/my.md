4. 飞控接入:你的架构天生好接,按三级走

你的系统对飞控的全部要求只有一句话:接收 10Hz 的机体速度指令 [vx,vy,vz,yaw_rate]。这是刻意的好设计——上层(DINO/WM/MPPI/V)完全不用动,换飞控只换"动作下发"这一层。

- 一级:PX4 SITL(仿真里换飞控)——ProjectAirSim 原生支持,你 sim_config 里现成有 scene_px4_sitl_wsl2.jsonc,src/px4/ 也有起飞/偏航/采图脚本。把 Simple Flight 换成 PX4 SITL、速度指令走 Offboard 模式,闭环脚本几乎只改连接层。这一步在仿真里就把"真飞控的时延/内环特性"提前吃掉,是实机前最有价值的一步。
